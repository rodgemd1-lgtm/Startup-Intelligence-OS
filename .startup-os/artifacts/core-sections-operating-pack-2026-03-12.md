# Core Sections Operating Pack

## Objective

Raise the shared OS layers from descriptive maturity to operating maturity by giving them reusable contracts, templates, and visible proof paths.

## Covered Sections

- Company Genome
- Evidence Graph
- Operator Console
- Memory & Evaluation
- Data Acquisition
- Persona & Agent Access
- Protocols & Processes

## Shared Build Rule

No core section can claim `10/10` until it has:

- a machine-readable contract
- a reusable template or operating packet
- a visible proof surface
- a refresh or review loop
- an explicit owner path

## Section Contracts

### Company Genome

Required operating behaviors:

- every company, project, decision, artifact, and run is file-backed
- orphan work is visible
- new major work creates a structured object, not only chat prose

### Evidence Graph

Required operating behaviors:

- every material recommendation can point to a named source bundle
- freshness and contradiction risk are reviewable
- source and proof paths are reusable

### Operator Console

Required operating behaviors:

- maturity is visible
- readiness is visible
- next-best actions are visible
- missing evidence is visible

### Memory & Evaluation

Required operating behaviors:

- every material run creates writeback
- a reusable review rubric exists
- failure patterns can be captured

### Data Acquisition

Required operating behaviors:

- manifests are declared
- jobs are resumable
- run summaries exist
- failure paths are visible

### Persona & Agent Access

Required operating behaviors:

- context bundles are explicit
- agents do not run blind on material asks
- access expectations are reusable across departments

### Protocols & Processes

Required operating behaviors:

- intake exists
- diagnosis exists
- outputs exist
- review exists
- escalation exists
