# Synthetic Run Review Framework

## Objective
Make synthetic runs first-class proof objects inside the benchmark library instead of treating them as transient tests.

## Review dimensions
- scenario quality
- grounding quality
- output usefulness
- failure visibility
- rubric alignment
- writeback completeness

## Scenario classes
- founder option comparison
- revenue and funnel forecast
- finance variance and runway stress test
- product workflow critique
- engineering implementation packet review
- training session rehearsal and facilitator-readiness check

## Eval test types
- rubric-graded output review
- benchmark comparison against named examples
- Monte Carlo scenario spread review
- failure injection and recovery review
- anti-pattern detection

## Synthetic-run review log
- run id
- scenario class
- source bundle used
- pass/fail
- confidence
- issues found
- writeback changes

## Promotion rule
No synthetic run counts as benchmark evidence until it has:
- a named scenario
- a named rubric
- a stored review log
- a writeback or follow-up action
