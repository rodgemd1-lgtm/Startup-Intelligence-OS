# Talent & Org Design Studio Operating Pack

## Objective

Turn Talent & Org Design Studio into a system for capability-gap-driven org design, role scorecards, hiring loops, and onboarding readiness.

## Current Gap

This is one of the weakest departments because the wrapper exists without enough working packets. It needs:

- role scorecard templates
- hiring loop templates
- onboarding plans
- org blueprint packets tied to capability gaps

## Operating System

### Core cadence

1. Capability-gap review
2. Role scorecard creation
3. Hiring loop launch
4. Onboarding plan and readiness review

### Required source stack

- department definition in `.startup-os/departments/talent-org-design-studio.yaml`
- live source manifest in `startup_os_talent_org_design_officials.yaml`
- capability-gap inputs from Jake/Susan and the maturity matrix
- resource and budget constraints from Finance & Operating Cadence Studio

### Required outputs

- org blueprint
- role scorecard
- hiring loop
- onboarding plan

## Scorecard

| Dimension | Question |
|---|---|
| Capability coverage | Does the role close a real capability gap? |
| Role clarity | Is the role scoped to outcomes, not generic duties? |
| Hiring readiness | Are the loop, evaluation, and calibration steps clear? |
| Onboarding readiness | Can the person become productive quickly with the assets provided? |

## Stage Gates

### Gate 1: Role validity
- gap named
- outcome named
- success criteria named

### Gate 2: Hiring loop quality
- interview sequence defined
- evaluation rubric defined
- panel and owner assigned

### Gate 3: Onboarding readiness
- first-30-day outputs defined
- required tools and source stack named

## Writeback

After every role or org-design cycle, write back:

- what the role is actually responsible for
- what was hard to evaluate
- which onboarding gaps slowed readiness
- what org-design assumptions changed
