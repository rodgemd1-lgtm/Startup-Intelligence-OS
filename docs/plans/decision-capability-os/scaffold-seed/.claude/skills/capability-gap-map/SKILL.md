---
name: capability-gap-map
description: Use when the user needs capability gaps mapped, a current-vs-target state analysis, maturity levels, ownership, or a capability roadmap.
---

Map capabilities in a structured way.

## Required sections
- capability area
- current state
- target state
- maturity delta
- gap description
- owner recommendation
- human vs agent split
- evidence or rationale
- sequence / prerequisites
- success criteria

## Rules
- capabilities must be concrete
- target state must be specific
- every gap should imply a build action
