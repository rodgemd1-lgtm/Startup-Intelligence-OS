# Strategic Frameworks Research — March 2026

## Topic 1: Future-Back Methodology
**Canonical source:** Innosight (Clayton Christensen's firm) — primary publisher. Strategos (Gary Hamel) — originated the term. Both use the same core logic.

### Steps
1. Current state value map (TSV analysis — where value lives today)
2. Future environment scan (customer JTBD evolution, industry boundary shifts, 5-10 year horizon)
3. Future state definition ("the future in which you intend to win" — single north star statement)
4. Backcasting: reverse-engineer milestones at 2-3 year intervals back to present
5. Migration Map: one-page strategic narrative connecting future → present (new offerings, channels, partnerships, capabilities, governance/culture requirements)
6. Near-term decision translation: resource reallocation, capability investments, portfolio reprioritization
7. Iterative loop: revisit vision + initiatives frequently, adjust as learning accumulates

### Key outputs
- Future state definition (5-10 year)
- Milestone ladder (2-3 year intervals)
- Migration Map (single-page)
- Near-term action list with accountability

## Topic 2: TRG Six Strategic Lenses
**Canonical source:** TRG Inc. learning hub (trg-inc.com/learning-hub/sixlenses)

1. **Urgency** — What happens if ignored? Time-sensitivity of action.
2. **Sequential Building Block** — What must happen first? Prerequisites and dependencies.
3. **Relationship Building** — Whose priorities does this serve? Alliance and stakeholder alignment.
4. **Feasibility** — Is this achievable with available resources and current environment?
5. **Team Commitment** — Where does genuine energy and buy-in exist?
6. **Potential Impact** — Which options have the greatest mission payoff?

### Application method
Each lens gets a color-coded dot on interventions. Team deliberates from each lens independently, then synthesizes. Use to generate discussion, surface divergence, and reach consensus on prioritization.

## Topic 3: Daniel Miessler PAI — True Architecture
**Canonical sources:** danielmiessler.com, github.com/danielmiessler/Personal_AI_Infrastructure, PAI v4.0.3

### 7 Architecture Components
1. Intelligence — model + scaffolding (skills, hooks, AI Steering Rules)
2. Context — 3-tier: Session Memory (30-day transcripts), Work Memory (ISC-tracked projects), Learning Memory (3,540+ signals)
3. Personality — 12 quantified traits (0-100), ElevenLabs TTS voice
4. Tools — 67 skills, 333 workflows, 200+ Fabric patterns, MCP integrations
5. Security — 4-layer defense (settings hardening, constitutional principles, pre-tool validation, safe execution patterns)
6. Orchestration — Hook system (17 hooks, 7 lifecycle events), 3-tier agent architecture
7. Interface — CLI-first, voice notifications, terminal tab management

### The Algorithm (inner operating loop)
OBSERVE → THINK → PLAN → BUILD → EXECUTE → VERIFY → LEARN
Key: Ideal State Criteria (ISC) = granular, binary, testable success conditions at each step

### TELOS layer (identity/context)
10 files: MISSION.md, GOALS.md, PROJECTS.md, BELIEFS.md, MODELS.md, STRATEGIES.md, NARRATIVES.md, LEARNED.md, CHALLENGES.md, IDEAS.md

### PAIMM maturity levels
- CH1-3: Chatbots (2022-2024)
- AG1-3: Agents (2025-2027) — current leading edge
- AS1-3: Assistants (2026+) — proactive, continuous advocacy, trusted companion

### Key principles
- "Scaffolding > Model Intelligence"
- "Solve Once, Reuse Forever" (UNIX composability)
- "Personalization over Prompting"
- "Context is Everything"
- "Convergence is Signal" (PAI, Claude Code, OpenCode all arrived at same patterns)
