|----------|-------------|------|
| [Flower (flwr)](https://github.com/adap/flower) | Most popular federated learning framework. Framework-agnostic. | 5k+ ★ on GitHub |
| [PySyft](https://github.com/OpenMined/PySyft) | Privacy-preserving ML. Remote data science on private data. | By OpenMined |
| [FATE](https://github.com/FederatedAI/FATE) | Industrial-grade federated learning framework | By WeBank |
| [TensorFlow Federated](https://github.com/google-parfait/tensorflow-federated) | Google's federated learning framework | By Google |
| [NVFlare](https://github.com/NVIDIA/NVFlare) | NVIDIA's federated learning SDK | Enterprise-grade |
| [OpenFL](https://github.com/securefederatedai/openfl) | Intel's open federated learning framework | Intel-backed |

### Key Concepts
- **Federated Learning** — Train models across distributed devices without sharing raw data
- **Differential Privacy** — Add mathematical noise to protect individual data points
- **Secure Multi-Party Computation (SMPC)** — Multiple parties compute on encrypted data jointly
- **Homomorphic Encryption** — Compute on encrypted data without decrypting it
- **Zero-Knowledge Proofs** — Prove computation correctness without revealing inputs
- **Trusted Execution Environments (TEEs)** — Hardware-level secure computation enclaves

## 10. Quick-Start Decision Framework

```
Want to BUILD a decentralized AI product?
├── AI inference on-chain → Ritual Infernet or Ora
├── Decentralized AI agents → Autonolas/OLAS or Morpheus
├── AI model marketplace → Bittensor subnet or SingularityNET
├── Privacy-preserving AI → Flower + PySyft
└── Decentralized training → Gensyn or io.net

Want to USE decentralized compute?
├── Cheapest cloud → Akash Network
├── GPU inference → io.net or Nosana
├── GPU training → Gensyn
└── Rendering + AI → Render Network

Want to INVEST/RESEARCH the space?
├── Blue chip → TAO, FET (ASI), RNDR
├── Infrastructure → AKT, IO, FLUX
├── Emerging → Ritual, Allora, 0G
└── Data → OCEAN, VANA, GRASS
```
