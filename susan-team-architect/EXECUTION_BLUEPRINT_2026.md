# Susan Execution Blueprint 2026

This is the operating blueprint for Startup Intelligence OS.

## North Star

Susan is the company architect. She should:
- determine what a company needs
- detect what expertise or datasets are missing
- route to the right studios and agents
- build new capabilities when the system does not yet have them
- write the work back into reusable memory

## Modes

- `quick`: fast answer, route, and compact next-step plan
- `deep`: deeper evidence pull, contradiction handling, and full planning
- `design`: design-session mode for UX, UI, motion, and moments of truth
- `foundry`: company-build mode for domains, studios, protocols, and execution tracks

## Phases

1. `Susan Core`
   - routing
   - evidence retrieval
   - mode selection
   - foundry blueprint generation
2. `Cognitive Studios`
   - design
   - marketing
   - social media
   - workout
   - recruiting
3. `Domain Packs`
   - TransformFit
   - Alex Recruiting
   - Oracle Health
4. `Expert Councils`
   - science
   - coaching
   - product experience
   - algorithms
   - recruiting intelligence
5. `Memory and Evaluation`
   - case libraries
   - anti-patterns
   - experiment logs
   - refresh jobs

## Execution Rule

Every new company or capability should follow:
1. problem framing
2. 5 Whys and JTBD
3. moments of truth map
4. evidence and dataset pull
5. studio routing
6. execution blueprint
7. memory writeback
