# Studio Open Source Research Map

This file is the high-signal map for the shared studio open-source layer. It is not a substitute for the raw source pages; it tells Susan and the studio agents what lives in the corpus and how to use it.

## Why This Exists

The studios need more than authored doctrine. They need a refreshable spine of public references that can be queried directly when a design, motion, discovery, or narrative decision depends on current implementation details or public best practice.

## Source Clusters

### 1. 2026 design craft and motion systems

- Apple `Liquid Glass`
  - use for navigation, glass-like chrome, and premium spatial layering
  - do not use as a blanket treatment on dense content surfaces
- Apple Human Interface Guidelines
  - baseline interaction and clarity rules for premium product surfaces
- WebKit scroll-driven animations and Safari 26 feature notes
  - native motion primitives for scroll timelines and modern platform support
- Chrome View Transitions
  - state continuity patterns for page and layout changes
- GSAP `ScrollTrigger` and `Flip`
  - cinematic motion and stateful layout choreography when native APIs are not enough

### 2. Cognitive discovery and problem framing

- Intercom JTBD material
  - supports 5 Whys and jobs-to-be-done discovery before layout or copy decisions
- NNGroup journey-mapping references
  - supports moments-of-truth mapping and friction diagnosis
- Design Council's evolved Double Diamond
  - keeps the studios from collapsing prematurely into execution before synthesis

### 3. Marketing and trust formation

- Zero Moment of Truth material
  - frames proof placement, search trust, and pre-click expectation management
- Discovery and mapping references
  - support belief-shift design, objection timing, and asset sequencing

## How The Studios Should Use This Layer

- `design-studio-director`
  - validate new design principles against current platform and motion capabilities
- `landing-page-studio`
  - decide when native CSS is enough versus when GSAP is justified
- `marketing-studio-director`
  - pull JTBD and moments-of-truth references before building messaging systems
- `deck-studio`, `memo-studio`, `whitepaper-studio`, `article-studio`
  - use ZMOT and discovery references to improve proof sequencing and belief change

## Query Patterns

- `liquid glass reduced motion`
- `view transitions state continuity`
- `gsap scrolltrigger narrative motion`
- `jobs to be done moments of truth`
- `zero moment of truth trust proof`
