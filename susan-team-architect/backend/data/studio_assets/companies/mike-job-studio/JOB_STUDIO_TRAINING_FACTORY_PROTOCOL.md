# Job Studio Training Factory Protocol

## Purpose

Define how Job Studio should build, use, evaluate, and improve strategist training assets and the supporting corpus.

## Protocol

1. **Start with the right track**
   - Track C for AI strategist foundations
   - Track B for Ellen and Gen Chat workflow enablement

2. **Attach the right corpus**
   - Job Studio memory
   - Oracle Health raw and extracted corpora
   - local repo harvests
   - behavioral-science and training docs

3. **Produce the full asset bundle**
   - deck
   - facilitator guide
   - learner handout
   - activity script
   - named examples
   - rubric

4. **Run quality gates**
   - source grounding
   - transcript clarity
   - transfer likelihood
   - conversational quality
   - reuse value

5. **Write back**
   - improved examples
   - failure patterns
   - revised prompts
   - revised facilitator notes
   - maturity deltas
   - context bundle usage notes

## Agent use

- Ellen leads the learner-facing experience.
- Jake and Susan own orchestration and department sequencing.
- Knowledge Engineer owns locator and corpus structure.
- AI Evaluation Specialist owns transcript and training quality review.
- Marketing Studio Director owns decks, narratives, and learner-facing asset quality.

## Access rule

Every material Job Studio training ask must explicitly attach the context bundles defined in `JOB_STUDIO_AGENT_ACCESS_PROTOCOL.md` before the team produces assets or recommendations.
