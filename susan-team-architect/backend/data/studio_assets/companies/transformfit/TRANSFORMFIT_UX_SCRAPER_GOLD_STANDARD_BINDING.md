# TransformFit UX Scraper Gold Standard Binding

## Status

TransformFit should treat `ux-design-scraper` as the gold-standard source for design workflow rigor and studio quality.

## What to inherit

- explicit phase-based design workflow
- critique depth
- token and design-system discipline
- output packaging quality
- compact operator-shell patterns for internal tools
- streaming/chat component clarity

## What not to inherit literally

- browser-extension chrome as a consumer app shell
- dense dashboard treatment for active workout sessions
- cold operator tone in coaching surfaces

## TransformFit-specific translation

### Landing and onboarding
- inherit strong narrative workflow
- use gold-standard critique and handoff rigor
- preserve TransformFit's coach-not-app doctrine

### Active workout session
- borrow structure, not shell visuals
- keep a static workout spine
- keep a dynamic coach thread
- use the gold standard for component states, loading feedback, accessibility, and implementation quality

### Internal coach and studio tools
- use the UX scraper shell much more directly
- sidebar + badge + workflow panel + queue metaphors are valid precedents here

## Decision rule

If there is tension between the UX scraper visual shell and TransformFit's consumer product needs:
- use the UX scraper for rigor, process, structure, critique, and execution quality
- use TransformFit doctrine for emotional tone, coaching feel, and workout-session calmness
