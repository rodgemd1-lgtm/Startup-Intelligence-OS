# Founder Memory Writeback Protocol

## Purpose

The foundry compounds only when launches, critiques, experiments, incidents, and design work create reusable memory.

## Required writeback events

- launch completed
- launch failed
- experiment completed
- decision reversed
- major studio critique
- incident or sev1/sev2 support event
- major positioning shift

## Required outputs

- short summary
- what changed
- why it mattered
- evidence or metrics involved
- reusable pattern
- anti-pattern if relevant
- owner
- linked company and track

## Destination types

- studio memory
- anti-pattern dataset
- operational protocol
- decision log
- experiment registry

## Founder rule

If it was painful enough to matter, it should be reusable enough to store.
