# Founder Foundry Domain Brief

Founder Intelligence OS is the product-facing workspace for Susan's startup foundry.

It should not be treated as a second backend or an alternate source of truth. The central backend in Startup Intelligence OS remains authoritative. Founder Intelligence OS exists to give Mike a repo-native surface for:

- founder-level planning
- company and domain creation
- reusable personas and frameworks
- studio operating systems
- launch, dashboard, and operating cadence design

## Foundry north star

Susan should be able to:

1. understand a new company or initiative
2. detect what is missing
3. retrieve the right evidence and examples
4. route the right experts and studios
5. stand up a new company genome and execution blueprint
6. write the work back into reusable memory

## Core systems that must exist

- Company genome
  - entities, dependencies, maturity, risk, budget, stage gates
- Evidence graph
  - sources, claims, contradictions, freshness, confidence
- Cognitive studios
  - diagnosis, critique, production, and memory
- Trust layer
  - privacy, security, accessibility, claims governance
- Operator console
  - KPI trees, dashboard specs, cadences, decisions, experiments
- Memory and evaluation
  - case libraries, anti-patterns, postmortems, eval harnesses

## Key design rule

The foundry should help build and run companies, not only describe them.
