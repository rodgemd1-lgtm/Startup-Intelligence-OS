# Founder Foundry Operating Cadence

## Weekly

- review company gap maps
- review stale datasets and low-coverage tracks
- review active experiments and launch risks
- update decision log and operator dashboard

## Biweekly

- refresh priority domain packs
- audit studio memory and postmortem writeback
- review trust and governance gaps for active companies

## Monthly

- rerun foundry blueprints for all flagship companies
- update maturity scores
- retire dead workstreams and low-value doctrines
- create new domain packs for emerging needs

## Quarterly

- evaluate the expert councils
- refresh official-source manifests
- audit evaluation rubrics
- review whether the founder repo and backend remain properly aligned

## Standard operating rule

If a company or studio lacks one of these, Susan should flag it:

- evidence
- owner
- cadence
- scorecard
- trust baseline
- writeback path
