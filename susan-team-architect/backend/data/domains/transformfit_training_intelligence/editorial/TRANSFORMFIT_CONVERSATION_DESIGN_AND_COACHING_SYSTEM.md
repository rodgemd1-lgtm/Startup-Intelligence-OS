# TransformFit Conversation Design and Coaching System

TransformFit’s product is not only the workout. It is the felt quality of the coaching relationship across sessions.

## Why Conversation Design Is A First-Class Function

- The coach is the product surface most users will remember.
- The conversation defines trust, not just tone.
- Distinct coach agents require differentiated voices, boundaries, and intervention styles.
- Generic “supportive AI” language destroys the premium coaching thesis.

## What Conversation Design Owns

- coach archetypes and role boundaries
- message taxonomy by session state
- motivational interviewing patterns
- microcopy for uncertainty, recovery, and adaptation
- silence rules and interruption rules
- escalation and defer-to-human language

## Coach Message Taxonomy

| Type | Purpose | Rule |
| --- | --- | --- |
| Instruction | tell the athlete what to do next | concrete, short, timed to action |
| Rationale | explain why a change or plan matters | only when explanation increases trust |
| Motivation | regulate effort or doubt | never generic hype |
| Adaptation | change the plan safely | explain the trigger in plain language |
| Reflection | reinforce identity or learning | use after the action, not before it |

## Motivational Interviewing In Product

Use MI as a pattern language, not a therapy simulation:

- ask open questions when context is missing
- affirm specific effort, not generic worth
- reflect what the athlete already said before redirecting
- summarize only when it proves understanding

Do not:

- manufacture intimacy too quickly
- ask intrusive questions without a clear coaching purpose
- mirror feelings so often that it becomes mechanical

## Conversation Rules By Session State

### Before Session

- reduce friction and uncertainty
- surface only one primary intention
- remind the athlete that the session fits their real life constraints

### During Session

- keep the spine stable
- coach messages must be short, timed, and useful
- avoid stacking multiple ideas during high exertion

### After Session

- reinforce progress and identity
- connect the session to the next return moment
- save one useful memory, not ten sentimental details

### Re-Entry After Missed Workouts

- reduce shame
- restore agency
- explain the smaller re-entry plan clearly
- do not punish absence with hyperactive streak logic

## Evaluation Rules

- rate every coach message for usefulness, trust, restraint, and state fit
- test whether athletes remember the guidance as helpful during hard moments
- track whether coach interventions speed or slow logging flow
- reject language that sounds warm but adds no action clarity

## Collaboration Model

- Conversation Designer defines voice, boundaries, and message taxonomy.
- Coaching Architecture Studio defines the state machine and memory rules.
- Workout Session Studio defines how coaching shows up in the interface.
- Flow and Freya validate motivation, adherence, and emotional-state fit.

## Build Recommendation

TransformFit should treat conversation design as a dedicated function by the time the product goes beyond prototype. Before that, the role must still exist in practice through an explicit owner with:

- message libraries
- coach personas
- evaluation rubrics
- warm-handoff rules
- escalation language
