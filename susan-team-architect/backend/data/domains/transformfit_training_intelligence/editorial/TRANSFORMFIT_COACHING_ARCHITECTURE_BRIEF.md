# TransformFit Coaching Architecture Brief

TransformFit needs a coaching-architecture studio because a generic AI assistant will not feel like a real training coach.

## What the studio should solve

- define coach voice and continuity
- separate instruction, motivation, adaptation, and reflection
- create silence rules for high-effort states
- explain plan changes without breaking flow
- preserve trust across missed sessions and re-entry
- build a relational memory system that feels human and bounded

## Coaching principles

- helpful brevity
- known-ness without overclaiming
- state-aware tone
- consistent logic across sessions
- trust boundaries and explicit uncertainty behavior
- context before content
- warmth with freshness decay

## Output expectations

- coach doctrine
- session-state message rules
- escalation rules
- trust-risk map
- experiment plan for perceived usefulness and trust
- personal-knowledge-map policy
