# TransformFit Workout Program Studio Brief

TransformFit needs a workout-program studio because workout generation is not the same as exercise recommendation. The program must manage progression, fatigue, confidence, adherence, substitution logic, time constraints, and the feeling that the system understands the user across multiple sessions.

## Studio mandate

- build programs that feel coached rather than random
- convert exercise science into usable mesocycles, sessions, swaps, and progressions
- protect users from novelty-heavy programming that feels intelligent but lacks continuity
- balance performance, recovery, confidence, and adherence

## What the studio should solve

### 1. Program architecture

- define blocks, mesocycles, microcycles, and weekly splits
- map main lifts, secondary work, accessories, and recovery work
- define progression rules and deload triggers

### 2. Exercise selection

- use real exercise catalogs with muscles, equipment, difficulty, and instruction data
- support substitution when equipment, pain, skill, or time changes
- distinguish identity-building lifts from utility lifts

### 3. Personalization logic

- beginner versus intermediate complexity
- hypertrophy, strength, fat-loss, and general-fitness tradeoffs
- volume tolerance and recovery constraints
- confidence-building first weeks

### 4. Adherence and return design

- the user must know why today’s session looks different from last week’s
- progress should be visible and framed as continuity, not just completion
- missed sessions should trigger graceful re-entry, not shame-heavy restart logic

## Required evidence

- physical-activity guidelines and public-health baselines
- open-access reviews on resistance-training volume, autoregulation, recovery, protein, and sleep
- user language from Reddit and app-store reviews
- structured exercise datasets for exercise selection and substitution

## Design standards for outputs

- every program must declare the goal, split, time budget, progression rule, substitution rule, and failure-adjustment rule
- every recommendation must distinguish evidence-backed logic from product judgment
- every plan must include a simplest-possible first week for confidence and compliance
