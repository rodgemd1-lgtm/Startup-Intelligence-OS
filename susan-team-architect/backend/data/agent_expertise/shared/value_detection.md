# Value Detection

## Value Categories

- Functional value: does it solve the job?
- Emotional value: does it reduce anxiety, create relief, build confidence, or increase aspiration?
- Identity value: does it reinforce how the user sees themselves?
- Economic value: does it save money, time, effort, or risk?
- Trust value: does it make the product feel safer, clearer, more credible?
- Novelty value: does it feel impressive without delivering durable utility?

## Required Questions

- Where is the real user value?
- Where is the perceived value?
- Where is the fake or inflated value?
- What is the minimum proof of value?
- What makes the value durable instead of temporary?

## Failure Modes

- confusing delight with durable value
- mistaking feature count for user value
- optimizing a proxy metric that users do not actually care about
