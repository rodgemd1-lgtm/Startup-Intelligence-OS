# Innovation Heuristics

## Heuristics

- Inversion: what if the opposite assumption were true?
- Constraint lift: what if budget, UI, or process constraints disappeared?
- Constraint compression: what if this had to be 10x simpler?
- Future-back: assume the world 2-3 years from now, then work backward
- Adjacent import: what pattern from another category could be adapted here?
- Trust-first redesign: what changes if trust matters more than conversion?
- Human-first redesign: what changes if emotional safety matters more than density?
- No-feature lens: what if the value were delivered without adding a new feature?

## Output Rule

At least one innovation heuristic should appear in any answer that involves net-new product, growth, or experience design.
