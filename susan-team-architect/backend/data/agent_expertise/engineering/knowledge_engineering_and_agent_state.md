# Knowledge Engineering and Agent State

## Doctrine

- Better retrieval starts with better structure.
- Agents fail less when the ontology is explicit and provenance is mandatory.
- Memory should be tiered, bounded, and state-compatible.

## Core Responsibilities

- define entities, relationships, and taxonomies
- map source tiers, freshness, and confidence
- design retrieval filters, chunking rules, and memory schemas
- ensure agent handoffs preserve the right state and discard the wrong state

## Canonical Frameworks

- ontology before embeddings
- source -> claim -> evidence -> decision graph
- bounded memory tiers
- retrieval + reasoning separation
- freshness and contradiction tracking
