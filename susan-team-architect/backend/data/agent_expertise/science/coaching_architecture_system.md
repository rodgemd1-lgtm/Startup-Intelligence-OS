# Coaching Architecture System

This pack defines how coaching systems should behave inside TransformFit-like products.

## Core Principle

A believable coach is state-aware, emotionally restrained, and useful under effort.

## Relationship layer

The coach should accumulate relational capital, not just recommendation quality.

Use:

- Love Maps for low-risk remembered detail
- Therapeutic Alliance for goals, tasks, and bond
- Perceived Responsiveness for feeling understood and cared for
- Relatedness for motivation support

## Context-before-content rule

Before asking for goals or prescribing action:

- ask about real-life context
- reflect the constraint back
- connect the coaching recommendation to that context

## Coaching layers

- instruction
- motivation
- adaptation
- reassurance
- reflection
- relational continuity

## Design Rules

- separate silence rules from speaking rules
- keep messages short inside active sessions
- explain changes when the plan adapts
- preserve continuity across days and missed sessions
- explicitly define escalation and uncertainty behavior
- let personal detail shape the plan more often than it appears in the copy
- use one warm personal cue sparingly rather than many

## Personal Knowledge Map

Track context in tiers:

- training context
- lifestyle context
- life details
- identity and values
- preference fingerprint

Each node should carry freshness and allowed-context rules.

## Uncanny Valley Risks

- stale-detail recall
- perfect-memory behavior
- sensitive-data ambush
- fake empathy without helpful action
- relational tone during cognitively overloaded moments

## Risks

- false personalization
- chatty noise
- advice without trust boundaries
- motivational tone detached from effort state
