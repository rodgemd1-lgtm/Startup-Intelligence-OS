# Training Research Studio System

This pack defines how the training-research studio should run evidence programs for workout design.

## Core doctrine

- Start from the decision the product must make, not the paper you hope to find.
- Exercise-science evidence must be translated into product rules and user-facing consequences.
- Structured catalogs, public-health guidance, and user-research language all matter; none is sufficient alone.

## Evidence ladder

1. public-health guidance
2. open-access systematic reviews and meta-analyses
3. newer pilot or implementation studies
4. structured exercise catalogs
5. user-language from reviews and communities

## Required outputs

- question tree
- evidence ladder
- contradiction map
- confidence note
- product implication
- program implication
- test recommendation

## Translation rules

- separate population-level guidance from product-level personalization
- separate evidence-backed claims from design judgment
- note where adherence logic is stronger than direct training-effect evidence

## Failure modes

- copying gym lore into product policy
- overfitting to one paper
- ignoring user language and implementation constraints
- presenting tentative evidence as hard certainty
