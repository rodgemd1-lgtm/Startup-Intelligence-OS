# Cognitive Studio Operating System

A world-class studio should not behave like a request queue.
It should behave like a cognitive system that senses, interprets, imagines, critiques, tests, remembers, and compounds.

## What makes a studio cognitive
The studio must do more than generate assets.
It must:
- perceive what is actually happening in the market, product, and customer experience
- diagnose where value is created or lost
- build hypotheses about what belief or behavior should change
- generate multiple concept directions before converging
- challenge its own ideas through critique and counterfactuals
- propose the cheapest validating experiments
- turn finished work into reusable memory, templates, and doctrine

## The eight-layer loop
1. Perception
   Gather customer language, behavior signals, screenshots, proof, competitor patterns, and business constraints.
2. Diagnosis
   Run 5 Whys, JTBD, moments of truth, objections, and friction mapping.
3. Value model
   Define:
   - user value
   - business value
   - emotional value
   - identity value
   - operational value
4. Concept generation
   Generate at least three routes:
   - conservative
   - differentiated
   - future-back or lateral-thinking route
5. Challenge loop
   Red-team each route for:
   - trust risk
   - proof gaps
   - usability risk
   - emotional mismatch
   - implementation cost
6. Experiment design
   State:
   - hypothesis
   - fastest test
   - success signal
   - failure signal
   - next decision
7. Production cascade
   Turn the winning direction into:
   - deliverable
   - template
   - derivative assets
   - implementation backlog
8. Memory
   Save:
   - what worked
   - what failed
   - examples
   - reusable principles
   - anti-patterns

## Mandatory reasoning modes
Every cognitive studio should be able to switch among:
- diagnosis mode
- concept mode
- skeptical-buyer mode
- operator mode
- future-back mode
- lateral-import mode
- red-team mode
- experiment mode

## Required data layers
1. Research layer
   - market evidence
   - customer language
   - user interviews
   - competitor references
2. Pattern layer
   - best-in-class examples
   - annotated screenshots
   - motion references
   - copy examples
3. Science layer
   - behavioral economics
   - cognitive psychology
   - service design
   - motivation and trust research
4. Studio memory layer
   - wins
   - failures
   - critique notes
   - templates
   - experiments and outcomes

## Mandatory studio protocols
- 5 Whys
- JTBD
- moments-of-truth map
- proof threshold map
- emotional outcome map
- objection map
- experiment plan
- postmortem and memory capture

## What changes in the studio output
Without cognition:
- prettier assets
- more content
- faster production

With cognition:
- better decisions
- stronger belief change
- cleaner reuse
- more differentiated concepts
- fewer generic outputs
- stronger compounding memory

## Implementation standard
Every studio output should include:
- diagnosed job
- target belief shift
- value map
- moments of truth
- concept options
- proof requirements
- experiment or validation plan
- reuse or derivative path

## Failure modes
- using the same reasoning pattern for every problem
- producing assets before diagnosing value
- converging on the first plausible concept
- no challenge loop
- no memory capture
- no distinction between real value and novelty value

## The standard
A cognitive studio should feel less like a content team and more like:
- an elite strategy group
- a research-informed design lab
- an editorial system
- a prototype-and-experiment engine
- a memory system that compounds over time
