# Company Analysis Framework

Susan's gap analysis identifies what a company needs and maps it to the optimal agent configuration.

## Complexity Scoring Rubric (1-10)

| Score | Description | Example |
|-------|-------------|---------|
| 1-2 | Standard, well-understood | Basic CRUD API |
| 3-4 | Some domain expertise needed | User authentication with MFA |
| 5-6 | Significant domain knowledge | Exercise programming with safety |
| 7-8 | Cross-domain integration | BE + exercise science + gamification |
| 9-10 | Novel, no established patterns | AI-driven behavioral adaptation |

## Risk Assessment Categories

1. **Technical Risk** — Can we build it? (infrastructure, APIs, data availability)
2. **Domain Risk** — Do we understand the problem? (exercise science, nutrition, sleep)
3. **Regulatory Risk** — Are there compliance requirements? (HIPAA, GDPR, FTC)
4. **Market Risk** — Will users want it? (validation, competition)
5. **Ethical Risk** — Could it cause harm? (body image, addiction, misinformation)

## Gap Analysis Template

For each capability gap:
- **Area**: What capability is missing
- **Current State**: What exists now (or nothing)
- **Ideal State**: What should exist
- **Complexity**: 1-10 score with justification
- **Agent Needed**: Which agent from the 22 templates
- **Risks**: Key risks from the 5 categories above
- **Cross-Portfolio Synergy**: Can this be reused across companies?

## Cross-Portfolio Synergy Matching

When analyzing a new company:
1. Check shared knowledge base for relevant existing chunks
2. Identify agents that have served similar roles in other companies
3. Flag reusable frameworks, templates, and data
4. Estimate cost savings from shared resources
