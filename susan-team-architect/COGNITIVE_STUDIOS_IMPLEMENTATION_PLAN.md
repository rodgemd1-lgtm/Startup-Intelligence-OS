# Cognitive Studios Implementation Plan
## How Susan's Design And Marketing Studios Become 25x Better

**Date:** 2026-03-07  
**Status:** Implementation blueprint

## Goal

Move the studios from:
- expert prompt personas
- asset and deliverable generation
- good doctrine with limited compounding memory

to:
- cognitive studios that diagnose value
- generate and challenge concepts
- design experiments
- create reusable templates
- compound from memory and case libraries

## What a cognitive studio is

A cognitive studio is a studio that can:
- sense what is happening
- explain why it matters
- identify where value is created or lost
- generate multiple options
- challenge its own ideas
- test before overcommitting
- turn output into reusable systems

The studio should behave less like a request queue and more like:
- a research-informed design lab
- a strategy group
- an editorial organization
- a memory engine

## The target architecture

### 1. Perception layer
Inputs:
- customer language
- screenshots
- workflows
- analytics
- competitors
- prior assets
- proof artifacts

### 2. Diagnosis layer
Required protocols:
- 5 Whys
- JTBD
- moments of truth
- objection map
- emotional outcome map
- proof threshold map

### 3. Value layer
Every studio brief should define:
- user value
- business value
- emotional value
- identity value
- operational value

### 4. Concept layer
Studios should not converge on the first plausible direction.
Generate:
- conservative route
- differentiated route
- future-back or lateral-thinking route

### 5. Challenge layer
Every concept should be red-teamed for:
- trust risk
- proof gaps
- sameness
- usability risk
- implementation burden
- second-order effects

### 6. Experiment layer
Every recommendation should include:
- hypothesis
- cheapest test
- pass signal
- fail signal
- next decision

### 7. Production layer
Turn the winning route into:
- deliverable
- implementation plan
- derivative assets
- templates

### 8. Memory layer
Save:
- examples
- anti-patterns
- winning language
- failed directions
- reusable modules
- measured outcomes

## Build sequence

### Phase 1: Operating system
- add shared cognitive studio doctrine
- upgrade design and marketing studio doctrine packs
- upgrade studio director agents
- enforce output contracts that include validation and reuse

### Phase 2: Data and memory
- build case libraries
- build screenshot and reference libraries
- build critique archive
- build pattern and anti-pattern libraries
- build experiment outcome logs

### Phase 3: Workflow
- require 5 Whys, JTBD, and moments-of-truth intake
- require concept options, not one-shot answers
- require challenge loop before final recommendation
- require template and derivative recommendations

### Phase 4: Evaluation
- score studio outputs for:
  - diagnosis quality
  - originality
  - proof quality
  - emotional precision
  - reuse value
  - experiment quality

### Phase 5: Compounding
- convert best outputs into reusable templates
- convert failed outputs into anti-patterns
- build studio-specific memory by company and domain

## Success criteria

The studios are meaningfully better when:
- they ask better questions before producing assets
- they produce multiple credible routes
- their outputs feel less generic and more ownable
- they recommend tests instead of just final answers
- they generate reusable templates and memory
- Susan can route problems to them as systems, not just specialists
