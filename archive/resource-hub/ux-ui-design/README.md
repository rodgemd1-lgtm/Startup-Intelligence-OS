## 6. UX Research Frameworks

| Framework | Description | When to Use |
|-----------|-------------|-------------|
| **Jobs-to-be-Done (JTBD)** | Focus on what job the customer is hiring your product to do | Product discovery, feature prioritization |
| **Design Sprint** (Google Ventures) | 5-day process for solving problems through prototyping and testing | New product/feature validation |
| **Double Diamond** | Discover → Define → Develop → Deliver framework by UK Design Council | End-to-end design process |
| **Lean UX** | Hypothesis-driven design with rapid experimentation | Startups iterating quickly |
| **HEART Framework** (Google) | Happiness, Engagement, Adoption, Retention, Task success | UX metrics and measurement |
| **System Usability Scale (SUS)** | 10-question standardized usability questionnaire | Quick usability benchmarking |
| **User Story Mapping** | Organize features by user journey and priority | Sprint planning, roadmapping |

### Key Books
- *The Design of Everyday Things* — Don Norman
- *Don't Make Me Think* — Steve Krug
- *Refactoring UI* — Adam Wathan & Steve Schoger (by the Tailwind team)
- *Sprint* — Jake Knapp (Google Ventures design sprint)
- *Lean UX* — Jeff Gothelf & Josh Seiden
