## 1. Positioning Framework (April Dunford Method)

### The 5 Components of Positioning

**Source:** April Dunford, "Obviously Awesome" (2024 expanded edition)

| Component | Question | Example |
|-----------|----------|---------|
| **Competitive Alternatives** | What would customers use if you didn't exist? | Spreadsheets, manual process, competitor X |
| **Unique Attributes** | What do you have that alternatives don't? | AI-powered, 10x faster, no-code |
| **Value** | What value do those attributes deliver? | Save 20 hours/week, reduce errors 90% |
| **Target Customers** | Who cares most about that value? | Engineering teams at Series A startups |
| **Market Category** | What market do you position in? | AI development platform |

### Positioning Statement Template
```
For [target customer]
Who [need/pain point]
[Product name] is a [market category]
That [key benefit]
Unlike [competitive alternative]
Our product [key differentiator]
```

## 2. Category Creation

### When to Create a Category
- Your product doesn't fit existing categories
- Existing categories are dominated by incumbents
- You can define the rules and win criteria
- You have a genuinely new approach

### Category Creation Playbook
1. **Name it**: Create a simple, memorable category name
2. **Define it**: What problems does this category solve?
3. **Contrast it**: How is it different from existing categories?
4. **Evangelize it**: Content, talks, analyst briefings
5. **Own it**: Be the defining example of the category

### Examples
| Company | Old Category | New Category |
|---------|-------------|-------------|
| **Figma** | Design tool | Collaborative design platform |
| **Notion** | Note-taking app | Connected workspace |
| **Linear** | Issue tracker | Modern project management |
| **Cursor** | Code editor | AI code editor |
| **Lovable** | No-code builder | AI software engineer |

## 3. Messaging Framework

### Messaging Hierarchy
```
Mission Statement (why you exist)
  ↓
Value Proposition (what you deliver)
  ↓
Key Messages (3 pillars)
  ↓
Supporting Proof Points (per pillar)
  ↓
Specific Use Cases / Stories
```

### Messaging Document Template
```markdown
# [Company] Messaging Framework

## Mission
One sentence: Why does your company exist?

## Value Proposition
For [audience]: One sentence that captures your unique value.

## Tagline
5-8 words that capture the essence.

## Key Messages

### 1. [Pillar 1: Speed]
Headline: [e.g., "Ship 10x faster with AI"]
Supporting: [How you deliver this]
Proof: [Metric, case study, or quote]

### 2. [Pillar 2: Quality]
Headline: [e.g., "Production-ready from day one"]
Supporting: [How you deliver this]
Proof: [Metric, case study, or quote]

### 3. [Pillar 3: Cost]
Headline: [e.g., "Replace your $50K agency"]
Supporting: [How you deliver this]
Proof: [Metric, case study, or quote]

## Elevator Pitch (30 seconds)
[Company] helps [audience] [achieve outcome] by [unique approach].
Unlike [alternatives], we [key differentiator], which means [value].

## Boilerplate (for press, about pages)
[Company] is [category description] that [what it does].
Founded in [year], [Company] [mission/vision].
[Traction statement]. Learn more at [website].
```

## 5. Storytelling Framework

### The Founder Story Structure
1. **The Origin**: What personal experience led to this?
2. **The Problem**: What broken thing did you discover?
3. **The Insight**: What did you see that others missed?
4. **The Solution**: How does your product fix it?
5. **The Vision**: What does the world look like if you succeed?

### Customer Story Structure
1. **Before**: What was their world like before your product?
2. **Trigger**: What made them look for a solution?
3. **Discovery**: How did they find you?
4. **Transformation**: What changed after adopting your product?
5. **Results**: Specific metrics and outcomes

### Content Narrative Formats
| Format | Length | Best For |
|--------|--------|----------|
| Twitter thread | 5-15 tweets | Viral reach, quick stories |
| LinkedIn post | 200-1,000 words | B2B audience, thought leadership |
| Blog post | 1,000-3,000 words | SEO, detailed stories |
| Case study | 500-1,500 words | Sales enablement |
| Video | 2-10 minutes | Demos, testimonials |
| Podcast | 30-60 minutes | Deep conversations |

## 6. Website Messaging

### Homepage Structure
| Section | Purpose | Key Element |
|---------|---------|-------------|
| **Hero** | Capture attention | Headline + subheadline + CTA |
| **Social proof** | Build trust | Logos, testimonials, metrics |
| **Problem** | Create resonance | "Do you struggle with X?" |
| **Solution** | Show your answer | Product screenshots + benefits |
| **Features** | Detail capabilities | 3-4 key features with visuals |
| **How it works** | Reduce complexity | 3-step process |
| **Testimonials** | Prove value | Quotes with names and photos |
| **Pricing** | Enable action | Clear tiers, highlight recommended |
| **CTA** | Drive conversion | Final call to action |

### Headline Formulas
- **Value**: "[Outcome] without [pain point]"
- **Speed**: "Build [thing] in [time], not [longer time]"
- **Comparison**: "The [better thing] for [audience]"
- **Question**: "What if you could [dream outcome]?"
- **Proof**: "[X] companies use [Product] to [outcome]"
