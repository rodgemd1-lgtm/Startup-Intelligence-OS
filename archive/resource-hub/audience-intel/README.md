# Audience Intelligence — Competitor Analysis & Market Research

> ICP definition, competitor mapping, market sizing, and buyer persona development.

---

## 1. Ideal Customer Profile (ICP) Framework

### ICP Definition Template
```
Company Attributes:
- Industry: [e.g., SaaS, fintech, healthcare]
- Company size: [e.g., 10-200 employees]
- Revenue: [e.g., $1M-$50M ARR]
- Tech stack: [e.g., uses React, AWS, Stripe]
- Growth stage: [e.g., Series A-B]

Buyer Persona:
- Title: [e.g., VP Engineering, CTO, Head of Product]
- Reports to: [e.g., CEO, CRO]
- Pain points: [specific problems they face daily]
- Goals: [what success looks like for them]
- Watering holes: [where they hang out online]
- Budget authority: [can they approve $X/month?]
```

### ICP Validation Process
1. Interview 20+ potential customers (Jobs-to-be-Done framework)
2. Analyze your best existing customers (if any)
3. Map common attributes across highest-value accounts
4. Score leads against ICP attributes (1-100)
5. Iterate ICP quarterly based on conversion data

## 2. Competitor Analysis Framework

### Competitive Intelligence Matrix
| Dimension | What to Track |
|-----------|-------------|
| **Product** | Features, pricing, integrations, UI/UX quality |
| **Positioning** | Messaging, tagline, value prop, category |
| **Distribution** | Channels, partnerships, marketplace presence |
| **Content** | Blog topics, SEO keywords, social engagement |
| **Funding** | Total raised, investors, valuation, burn rate |
| **Team** | Key hires, LinkedIn profiles, org structure |
| **Reviews** | G2, Capterra, TrustRadius scores and themes |
| **Technical** | Tech stack (BuiltWith), API, performance |

### Competitor Research Tools
| Tool | Purpose | Pricing |
|------|---------|---------|
| **SimilarWeb** | Traffic, audience, channels | Free tier + paid |
| **BuiltWith** | Tech stack detection | Free tier |
| **Ahrefs** | SEO, backlinks, content | $99/mo+ |
| **Crunchbase** | Funding, team, news | Free tier + paid |
| **G2** | Reviews, comparisons | Free to view |
| **Apify** | Web scraping at scale | Usage-based |
| **Exa AI** | Semantic search for intel | API-based |
| **Owler** | Company intelligence | Free tier |
| **SpyFu** | PPC competitor intel | $39/mo+ |

### How to Track 47+ Competitors
1. Create a Google Sheet / Notion database with all competitors
2. Categorize: Direct, Indirect, Adjacent, Emerging
3. Set up Google Alerts for each competitor name
4. Monitor their Product Hunt launches, blog posts, funding
5. Track feature launches via changelog/release notes
6. Review quarterly: Who's gaining? Who's losing? Why?

## 3. Market Sizing (TAM/SAM/SOM)

### Bottom-Up Approach (Preferred by Investors)
```
SOM = (# reachable customers) × (annual contract value)
SAM = (# potential customers in your segment) × ACV
TAM = (# total potential customers globally) × ACV
```

### Example for AI SaaS Tool
```
TAM: 500,000 companies using AI tools × $1,200/yr = $600M
SAM: 50,000 mid-market SaaS companies × $1,200/yr = $60M
SOM: 2,000 reachable companies × $1,200/yr = $2.4M (Year 1)
```

## 4. Buyer Persona Development

### Persona Template
```
Name: "Technical Tom" (VP Engineering)
Demographics: 30-45, 8+ years experience, $180-250K salary
Company: 50-200 person SaaS startup, Series A-B
Goals:
  - Ship features 2x faster
  - Reduce engineering costs
  - Maintain code quality at scale
Pain Points:
  - Can't hire fast enough
  - Technical debt slowing releases
  - Too much context-switching
Information Sources:
  - Hacker News, Twitter/X, tech blogs
  - Engineering podcasts, YouTube channels
  - GitHub trending, Stack Overflow
Decision Process:
  - Evaluates tools independently first
  - Gets team buy-in through trial
  - Approves purchases < $500/mo
Objections:
  - "Is it secure enough?"
  - "Will it integrate with our stack?"
  - "What's the learning curve?"
```

## 5. Voice of Customer (VoC) System

### Data Collection
| Channel | What to Capture |
|---------|----------------|
| **Support tickets** | Pain points, feature requests, friction |
| **Sales calls** | Objections, buying triggers, competitor mentions |
| **Reviews** (G2, etc.) | What they love, what they hate |
| **Surveys** (NPS, CSAT) | Satisfaction trends, themes |
| **Social listening** | Brand mentions, sentiment, trending topics |
| **Community** | Questions, workarounds, feature ideas |

### Analysis Framework
1. Tag all feedback by theme (feature, bug, UX, pricing, etc.)
2. Quantify: How many people mention each theme?
3. Score impact: Revenue risk × frequency
4. Prioritize: High impact + high frequency = build next
