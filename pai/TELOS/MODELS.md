# Models

Mental models and decision-making frameworks Mike uses regularly.

## Decision Models

### Strategos
Strategic decision framework. For every major choice: define the objective, map the terrain, identify options, assess risks, choose, execute, debrief.

### Jordan Voss Test
"Can I identify the one move today in <30 seconds?" If the answer is no, the system is too complex.

### FQR Ratio (Feature-Quality-Revenue)
Before building any feature: Does it improve quality? Does it drive revenue? If neither, it's scope creep.

### Debt Circuit Breaker
Technical debt score > 30 = no new features until debt is paid down. Non-negotiable.

### Reversibility Check
Is this decision reversible? If yes, move fast. If no, research more, get a second opinion, sleep on it.

## Operational Models

### 25x Efficiency
Every system should enable one person to do the work of 25. If it doesn't approach that leverage, rethink the architecture.

### Research-First Pipeline
New company or initiative? Run the 5-phase research pipeline before any code is written. Scope -> Research -> Ingest -> Team -> Score.

### Context Health Monitor
Track context scatter (files touched), error rate, and scope drift in every session. ORANGE = slow down. RED = handoff immediately.

## Evaluation Models

### Miessler PAI Maturity
Score PAI capability on a 0-100 scale across: memory, autonomy, channels, learning, security, proactivity. Current: 34/100. Target: 95/100.

### 6-Dimension Project Assessment
Rate any project across: Research, Team, Tech, Strategy, Capability, Governance. Each dimension 0-10. Total < 30 = not ready.
