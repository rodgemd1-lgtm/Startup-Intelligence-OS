# Wrong

Things I was wrong about. Honesty about mistakes accelerates learning.

## Technology
- **"Hermes would work"** — Built a 5-phase bot system that scored 34/100. Three disconnected bots, no unified memory, no autonomy. Should have started with architecture, not features.
- **"Just code it up"** — Skipping the research phase cost more time than it saved. Every time. The R->P->E->L->D process exists because of this lesson.
- **"More features = more value"** — Added complexity without adding capability. 99K memories that can't be retrieved usefully is worse than 1K well-structured ones.
- **"I can build it all in one session"** — Context degrades. Sessions have natural limits. Fighting it produces bugs, not features.

## Business
- **"Technical excellence sells itself"** — It doesn't. Oracle Health taught me that change management and stakeholder alignment matter more than the best architecture.
- **"Solo founder means doing everything"** — Solo founder means building leverage, not doing more work. That's what agents are for.

## Process
- **"Plans slow you down"** — Plans save time. Every plan that felt like overhead prevented at least one expensive rework.
- **"I'll remember the context"** — No, I won't. And neither will the AI. Write it down. HANDOFF.md exists because of this.
